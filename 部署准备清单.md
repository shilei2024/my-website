# 部署准备清单

## 1. 检查网站文件

确保以下文件都存在：
```
D:\Company\
├── index.html          ✓ 主页面
├── style.css          ✓ 样式文件
├── script.js          ✓ JavaScript文件
├── assets/            ✓ 资源文件夹
│   └── images/        ✓ 图片文件夹
├── 修改指南.md        ✓ 修改说明
├── 修改示例.html      ✓ 修改示例
└── 响应式测试指南.md  ✓ 测试指南
```

## 2. 测试网站功能

在本地测试以下功能：
- [ ] Cookie提示栏可关闭
- [ ] 导航菜单正常（桌面和移动端）
- [ ] 轮播图自动播放和手动控制
- [ ] 产品卡片悬停效果
- [ ] 行业应用标签页切换
- [ ] 所有链接可点击
- [ ] 响应式设计正常

## 3. 优化建议（可选）

### 3.1 图片优化
- 压缩图片文件大小
- 使用适当的格式（JPG用于照片，PNG用于Logo）
- 确保图片尺寸合适

### 3.2 代码优化
- 删除不必要的注释
- 确保没有调试代码（如console.log）
- 检查所有链接是否正确

### 3.3 SEO优化
- 确保每个页面有唯一的<title>
- 添加<meta description>描述
- 使用语义化HTML标签

## 4. 最终检查清单

### 4.1 内容检查
- [ ] 公司名称正确
- [ ] 联系信息正确
- [ ] 产品描述准确
- [ ] 版权信息正确
- [ ] 所有文字无错别字

### 4.2 功能检查
- [ ] 所有JavaScript功能正常
- [ ] 表单验证正常（如果有）
- [ ] 移动端菜单正常
- [ ] 轮播图正常

### 4.3 兼容性检查
- [ ] 在Chrome浏览器正常
- [ ] 在Firefox浏览器正常
- [ ] 在Safari浏览器正常（如果有Mac）
- [ ] 在Edge浏览器正常

## 5. 部署前备份

建议备份以下文件：
1. 整个项目文件夹复制到其他位置
2. 或者使用Git进行版本控制

## 6. 选择部署平台

根据您的需求选择：
- **Vercel**：最简单，推荐新手使用
- **Netlify**：功能丰富，同样简单
- **GitHub Pages**：适合开发者，需要Git知识

## 7. 域名准备（可选）

如果您有自己的域名：
- 确保域名已购买并可以管理DNS
- 准备好域名解析设置

## 8. 部署后测试

部署完成后需要测试：
- [ ] 网站可通过公网访问
- [ ] HTTPS证书正常（锁图标）
- [ ] 所有功能正常
- [ ] 加载速度合理

完成以上检查后，您的网站就准备好部署了！
