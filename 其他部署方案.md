# 其他部署方案

## 方案B：使用Netlify部署

### Netlify优点：
- 同样简单易用
- 免费套餐充足
- 表单处理功能
- 强大的CMS支持

### 部署步骤：

#### 步骤1：注册Netlify账号
访问：https://app.netlify.com/signup

#### 步骤2：部署网站
1. 登录Netlify控制台
2. 点击"Add new site" → "Deploy manually"
3. 将`Company`文件夹拖入上传区域
4. 等待部署完成

#### 步骤3：访问网站
部署后获得URL：`https://your-site-name.netlify.app`

#### 步骤4：自定义域名
1. 站点设置 → "Domain management"
2. 添加自定义域名
3. 按照提示配置DNS

## 方案C：使用GitHub Pages部署

### GitHub Pages优点：
- 完全免费
- 与GitHub集成
- 适合开发者

### 前提条件：
- 需要有GitHub账号
- 需要基本Git知识

### 部署步骤：

#### 步骤1：创建GitHub仓库
1. 登录GitHub
2. 点击"New repository"
3. 仓库名：`my-company-website`
4. 选择"Public"（公开）
5. 点击"Create repository"

#### 步骤2：上传文件到GitHub
**方法A：使用Git命令行**
```bash
# 初始化Git仓库
cd D:\Company
git init
git add .
git commit -m "Initial commit"

# 连接到GitHub
git remote add origin https://github.com/你的用户名/my-company-website.git
git branch -M main
git push -u origin main
```

**方法B：使用GitHub Desktop**
1. 下载GitHub Desktop：https://desktop.github.com
2. 添加本地仓库
3. 提交更改
4. 推送到GitHub

**方法C：网页直接上传**
1. 在GitHub仓库页面点击"Add file" → "Upload files"
2. 拖入`Company`文件夹所有文件
3. 点击"Commit changes"

#### 步骤3：启用GitHub Pages
1. 仓库设置 → "Pages"
2. 分支选择：`main` 或 `master`
3. 文件夹选择：`/(root)`
4. 点击"Save"

#### 步骤4：访问网站
URL格式：`https://你的用户名.github.io/my-company-website`

## 方案D：传统虚拟主机

### 适用情况：
- 已有虚拟主机服务
- 需要PHP/数据库支持
- 企业级需求

### 部署步骤：

#### 步骤1：准备FTP客户端
推荐软件：
- FileZilla（免费）
- WinSCP（免费）
- Cyberduck（免费）

#### 步骤2：获取FTP信息
从主机商获取：
- FTP服务器地址
- 用户名
- 密码
- 端口（通常是21）

#### 步骤3：上传文件
1. 连接FTP服务器
2. 上传`Company`文件夹所有文件到`public_html`或`www`目录
3. 确保文件权限正确（644 for files, 755 for folders）

#### 步骤4：访问网站
通过您的域名访问：`http://yourdomain.com`

## 故障排除

### 常见问题1：404错误
**可能原因**：文件路径错误
**解决方法**：
- 检查`index.html`是否在根目录
- 检查资源文件路径
- 清除浏览器缓存

### 常见问题2：混合内容警告
**可能原因**：HTTP资源加载
**解决方法**：
- 将所有资源链接改为HTTPS
- 使用相对路径
- 更新外部资源链接

### 常见问题3：CORS错误
**可能原因**：跨域资源加载
**解决方法**：
- 确保API支持CORS
- 使用代理服务器
- 避免跨域请求

### 常见问题4：性能问题
**可能原因**：资源过大或过多
**解决方法**：
- 压缩图片
- 合并CSS/JS文件
- 使用CDN
